# Code of Conduct

## Our commitment

NeuroAgent is an open, collaborative project. Everyone who participates — as a contributor, reviewer, issue reporter, or user — helps shape the tone of the community. This document sets the baseline expectations for that participation.

We commit to keeping NeuroAgent a welcoming, professional space where people of any background, experience level, or identity can contribute without fear of mistreatment.

## Expected behaviour

When participating in any project space — issues, pull requests, discussions, reviews, or direct communication about the project — please:

- **Be respectful.** Assume good faith. Address the idea or the code, not the person.
- **Be constructive.** Feedback should help the recipient improve, not put them down.
- **Be patient.** Maintainers and contributors are volunteers; response times vary. New contributors are still learning the codebase — that's expected, not a problem.
- **Be clear.** Ask for what you need (a review, a decision, more context) rather than hinting. If something is unclear, ask.
- **Credit others.** If you build on someone else's work, acknowledge it.
- **Accept feedback gracefully.** Disagreement is fine; dismissiveness is not. If a reviewer asks for a change, either address it or explain why you'd rather not.

## Unacceptable behaviour

The following are not acceptable anywhere in project spaces:

- Personal attacks, name-calling, or belittling language directed at contributors or users.
- Discrimination or exclusion based on personal characteristics.
- Intimidation or sustained disruption of discussions, issues, or reviews.
- Publishing private information (of anyone) without consent.
- Conduct that would reasonably be considered inappropriate in a professional software engineering setting.

## Scope

This Code applies to all project-managed spaces, including:

- The GitHub repository (issues, pull requests, discussions, code review comments, commit messages).
- Any community channels the project may add in future (chat, forums, mailing lists).
- Project-related communication elsewhere when a participant is identifying themselves as a representative of NeuroAgent.

## Reporting

If you experience or observe behaviour that violates this Code, report it to the maintainers by email at **hamzaahmedsiddiqui03@gmail.com**. Reports will be read promptly and handled confidentially. Please include:

- What happened (and a link, if it's on GitHub).
- When it happened.
- Any context you think is relevant.

You do not need to be the target of the behaviour to report it.

## Enforcement

Maintainers are responsible for interpreting and enforcing this Code. Depending on severity and pattern, responses may include:

1. **A private reminder** — a note clarifying the expectation, with no public record.
2. **A public correction** — a maintainer asks for a change in behaviour in the thread.
3. **A temporary restriction** — the participant is asked to step away from a specific space or issue for a defined period.
4. **A permanent ban** — for severe or repeated violations, the participant loses the ability to interact with the project.

Maintainers who do not uphold this Code in good faith may face the same consequences as other participants.

## A note on disagreement

Technical disagreement is not a Code of Conduct issue. Healthy projects argue about architecture, dependencies, and trade-offs all the time. The line is *how* you argue: a sharp critique of an approach is welcome; a sharp critique of the person who proposed it is not.
